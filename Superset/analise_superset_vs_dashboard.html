🔍 O que é o Superset?

O sistema que você recebeu acesso é um Apache Superset conectado via ClickHouse (data warehouse) diretamente à Phoenix (plataforma do JogoGrande). Isso significa que os dados são processados via API em tempo real, sem precisar do arquivo XLSX diário que recebemos por email.

Ele puxa dados de 4 tabelas principais do ClickHouse (project_0): client_balance, bet_change (sportsbook), silver.casino, e payment. São dados granulares por transação — muito mais detalhados do que o resumo diário que temos hoje.

🆕 Dados TOTALMENTE NOVOS que o Superset traz

Esses são dados que não existem em lugar nenhum no nosso dashboard atual. São as maiores oportunidades de inteligência.

💰 Saldo Real-Time dos Jogadores

Saber quanto dinheiro (real + bônus) está nas contas dos jogadores AGORA. Permite prever cash-out e monitorar liquidez. Nenhuma outra fonte tem isso.

🎮 Performance por Jogo/Provider

Sabemos agora QUAL jogo, de QUAL provider, gera mais GGR. Podemos identificar jogos "vampiros" (RTP alto = prejuízo) e jogos lucrativos.

📊 RTP (Return to Player) %

Métrica crítica que mostra quanto % do turnover volta pro jogador. RTP alto demais = prejuízo. Essencial pra negociar com providers de casino.

⚽ Análise por Esporte/Torneio/Evento

Dados detalhados de CADA aposta: qual esporte, qual campeonato, qual jogo, qual mercado. Permite entender perfil de apostador.

⏳ Apostas Pendentes (Pending)

Valor total de apostas UNSETTLED — dinheiro em risco que ainda não foi liquidado. Essencial pra gestão de risco em tempo real.

📈 ARPU (Revenue per User)

Receita média por jogador ativo. Tanto pra Casino quanto Sportsbook. Métrica essencial de unit economics.

💳 Taxa de Sucesso de Pagamento

% de transações PAID vs tentativas totais. Permite identificar problemas com gateway, método de pagamento específico, etc.

🔍 Método de Pagamento

Breakdown por PIX, Boleto, Cartão etc. Permite otimizar checkout e entender preferências dos jogadores.

📊 Cohort Analysis Nativa

5 pivot tables de cohort: por Casino, Sport, Payment, Usuários, e Número de Pagamento. Análise de retenção por mês de registro.

🏷️ BTAG (Affiliate Tag)

Tag de afiliado presente em TODOS os datasets (Casino, Sport, Payment). Permite cruzar performance do jogador com a fonte de tráfego.

🌍 País do Cliente

client_country em todos os datasets. Permite segmentar performance por geografia.

✅ Status de Verificação

client_is_verified — flag de KYC. Permite comparar comportamento de jogadores verificados vs não verificados.

🎯 Recomendações Estratégicas

1. NÃO SUBSTITUIR — COMPLEMENTAR

O Superset NÃO substitui nosso dashboard. Ele tem Casino/Sport/Payment detalhados, mas NÃO tem: Registros (NR), NGR calculado, dados Smartico (afiliados), dados PSP (conciliação). A estratégia é INTEGRAR os dois.

2. Incorporar RTP + GGR% Imediatamente

RTP (Return to Player) é a métrica mais crítica que ganhamos. Um jogo com RTP > 97% pode estar sangrando dinheiro. Precisa estar no dashboard PRINCIPAL.

3. Criar Alerta de Pending Bets

Valor de apostas não liquidadas é RISCO em aberto. O agente analista deve monitorar e alertar quando ultrapassar threshold definido por vocês.

4. Dashboard de Casino por Provider/Jogo

Criar aba específica com ranking de providers e jogos por GGR, RTP e volume. Isso permite negociar com providers e remover jogos não lucrativos.

5. Cross-Reference BTAG com Smartico

O BTAG do Superset + dados Smartico permite fechar o loop: qual afiliado → quais jogadores → quanto gastaram em Casino/Sport → qual o ROI real do afiliado.

6. Validação Cruzada de Números

PRIORIDADE: Cruzar os valores de depósitos/saques do Superset com o que temos do Phoenix XLSX e do PSP CSV. Se houver divergência, temos um problema de integridade de dados.
